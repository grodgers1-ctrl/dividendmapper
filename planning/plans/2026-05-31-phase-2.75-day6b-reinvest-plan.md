# Phase 2.75 Day 6B — Reinvest Recommender (hygiene-framed) + ex-div data layer

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans. Strict TDD: failing test → run → impl → run → commit, one logical unit per commit. Steps use checkbox (`- [ ]`) syntax.

**Goal:** When a holding goes ex-dividend within 5 days, show a **diversification / income-hygiene** card on `/app/portfolio` that estimates the upcoming payment and suggests where the cash could go to keep the portfolio spread out — driven by Quality score **and** position concentration, never by "outperformance."

**Architecture:** The portfolio page is already a server component that computes holdings, UK-merged quotes, per-holding `concentration` weights, `scoresByTicker`, and FX rates. The only missing input is ex-dividend dates, which the nightly cron will now persist into `equity_scores` (additive — **no score-compute change**). The card is assembled **server-side** from those pieces (same server-join precedent that replaced `/api/scoring/holdings` in Day 6A) and passed as props. The only network endpoint is `POST /api/reinvest/log` for outcome logging. Ranking reuses the Day-5 `0.6·quality + 0.4·income` engine, multiplied by a new `concentrationFactor`.

**Tech Stack:** Next.js 16.2.4 (App Router, server components), React 19.2.4, Tailwind v4, Supabase (RLS: score tables public-read, log table self-RLS), Vitest + RTL/jsdom, FMP `dividends-calendar`.

## Recalibration notes (draft Day-6B scope → resolved design, 2026-05-31)

Grounded in live data + two design decisions confirmed with Glenn:

1. **Ex-div source = persist in the nightly cron** (Glenn's decision A; the alternative was live-FMP-at-request). The cron already pulls the market-wide `dividends-calendar` once per run; we persist each ticker's *soonest upcoming* ex-div into three new `equity_scores` columns. **Scores are byte-identical** — this honors "do not change what the cron computes." Cost: the card cannot light up until a successful post-deploy cron run populates the dates.

2. **Card framing = diversification / income-hygiene, driven by Quality AND concentration** (Glenn's decision B). Redirect/alpha is disproven and dead (see `project_buy_score_backtest_artifact`): **no "outperform / better total return" language anywhere.** The existing `lib/reinvest` math is fine; the WHY and the copy are hygiene. Concentration (`lib/portfolio/concentration.ts`) demotes candidates at/over the 20% cap.

3. **Both draft GET routes dropped; server-join instead.** `GET /api/dividends/upcoming` and `GET /api/reinvest/suggestions` are replaced by server-side assembly on `page.tsx`, exactly as Day 6A dropped `/api/scoring/holdings`. "Show more" reveals already-passed ranks 6–10 (no fetch). Only `POST /api/reinvest/log` remains (logging must be a route).

4. **Buttons are log-only — no outbound links** (no broker sync until Phase 3). Per-candidate **"Use this idea"** logs `accepted` + that ticker and shows an inline confirmation; card-level **"Not now"** logs `dismissed` + collapses for the session; **"Show more"** reveals 6–10 and logs `shown_only`. We never fake a "Buy now."

5. **Trigger selection.** The soonest holding with ex-div ≤5 days (tie → larger estimated GBP payment). An **unscored** holding (e.g. VOD.L) may be a *trigger* but never a *candidate*. The card is **suppressed when there are no eligible candidates**.

```ts
import type { FmpCalendarDividend } from "./fmp-client";

// The market-wide dividends-calendar contains every ticker's upcoming ex-div
// dates. Find this ticker's soonest one on or after `todayIso`. ISO date strings
// (YYYY-MM-DD) sort lexicographically, so plain string compare is correct.
export function nextUpcomingDividend(
  calendar: ReadonlyArray<FmpCalendarDividend>,
  ticker: string,
  todayIso: string,
): FmpCalendarDividend | null {
  let best: FmpCalendarDividend | null = null;
  for (const row of calendar) {
    if (row.symbol !== ticker) continue;
    if (row.date < todayIso) continue;
    if (best === null || row.date < best.date) best = row;
  }
  return best;
}
```

- [ ] **Step 3: Implement**
```ts
// Reinvest hygiene weighting. Below the concentration threshold, reward
// under-weight positions (up to +15% at weight 0, tapering to neutral at the
// threshold). At or above the threshold, hard-penalise (×0.5): adding there
// worsens concentration. Pure + bounded for easy testing.
export function concentrationFactor(weight: number, threshold = 0.2): number {
  if (!Number.isFinite(weight) || weight < 0) return 1.15;
  if (weight >= threshold) return 0.5;
  return 1.15 - 0.15 * (weight / threshold);
}
```

  Add the `diversificationNote` helper in the same file (one clause; hygiene, never alpha; no em dashes):
```ts
function diversificationNote(args: {
  weight: number | null;
  threshold: number;
  candidateSector: string | null;
  triggerSector: string | null;
}): string | null {
  const { weight, threshold, candidateSector, triggerSector } = args;
  if (weight !== null && weight >= threshold) {
    return "already a large position";
  }
  if (
    candidateSector &&
    triggerSector &&
    candidateSector.toLowerCase() !== triggerSector.toLowerCase()
```

```ts
import { NextResponse } from "next/server";
import { createSupabaseServerClient } from "@/lib/supabase/server";

export const runtime = "nodejs";
export const dynamic = "force-dynamic";

const ACTIONS = ["accepted", "dismissed", "shown_only"] as const;
type Action = (typeof ACTIONS)[number];
const DATE_RE = /^\d{4}-\d{2}-\d{2}$/;
const TICKER_RE = /^[A-Z0-9.\-]{1,12}$/;

export async function POST(req: Request) {
  const supabase = await createSupabaseServerClient();
  const { data: claims } = await supabase.auth.getClaims();
  const userId = (claims?.claims?.sub as string | undefined) ?? null;
  if (!userId) return NextResponse.json({ error: "unauthorized" }, { status: 401 });

  let body: unknown;
  try { body = await req.json(); } catch { return NextResponse.json({ error: "invalid_input" }, { status: 400 }); }
  const b = (body ?? {}) as Record<string, unknown>;

  const triggerHoldingId = typeof b.triggerHoldingId === "string" ? b.triggerHoldingId : "";
  const triggerExDivDate = typeof b.triggerExDivDate === "string" ? b.triggerExDivDate : "";
  const userAction = b.userAction as Action;
  const suggestedTickers = Array.isArray(b.suggestedTickers)
    ? b.suggestedTickers.filter((t): t is string => typeof t === "string" && TICKER_RE.test(t.toUpperCase()))
    : [];
  const userActionTicker =
    typeof b.userActionTicker === "string" && TICKER_RE.test(b.userActionTicker.toUpperCase())
      ? b.userActionTicker.toUpperCase()
      : null;

  if (!triggerHoldingId) return NextResponse.json({ error: "invalid_input" }, { status: 400 });
  if (!DATE_RE.test(triggerExDivDate)) return NextResponse.json({ error: "invalid_input" }, { status: 400 });
  if (!ACTIONS.includes(userAction)) return NextResponse.json({ error: "invalid_input" }, { status: 400 });
  if (suggestedTickers.length === 0) return NextResponse.json({ error: "invalid_input" }, { status: 400 });

  const { error } = await supabase.from("reinvest_suggestions_log").insert({
    user_id: userId,
    trigger_holding_id: triggerHoldingId,
    trigger_ex_div_date: triggerExDivDate,
    suggested_tickers: suggestedTickers,
    user_action: userAction,
    user_action_ticker: userAction === "accepted" ? userActionTicker : null,
    acted_at: userAction === "shown_only" ? null : new Date().toISOString(),
  });
  if (error) return NextResponse.json({ error: "write_failed" }, { status: 500 });
  return NextResponse.json({ ok: true });
}
```
